## Fix: setup self-test stuck on “Cursor is working”

After **StayOn: Set Up**, the panel sometimes opened and showed **“Cursor is working”** even though no Agent was running. A Cursor reload fixed it.

**Cause:** Setup runs a hook self-test that sends `busy_start` to verify the bridge, but never sent `busy_end` / `session_end`, and it triggered the same panel flow as a real Agent wait.

**v0.1.3:** Setup self-test now:
- Skips opening the panel and starting a task session
- Sends `session_end` to reset bridge state after verification
- Verifies success via `hook.log` (works with inbox fallback on Windows)

After setup, the panel correctly shows **“Waiting for agent…”** until you submit a real Agent prompt.

**Unchanged:** **StayOn: Test Hook Bridge** still opens the panel on purpose so you can confirm the busy UI manually.
