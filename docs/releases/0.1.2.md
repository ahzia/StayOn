# StayOn v0.1.2

**Cursor extension ¬∑ Windows + macOS beta**

Install: `extension/stayon-0.1.2.vsix` ‚Üí Extensions ‚Üí Install from VSIX ‚Üí reload ‚Üí **StayOn: Set Up**

---

## Highlights

**Fixes Agent panel not opening on Windows** when Cursor‚Äôs hook sandbox blocks HTTP to the local bridge. Hooks now fall back to a file-based inbox; the extension polls and applies events the same way as HTTP.

If **StayOn: Test Hook Bridge** worked but real Agent prompts did not, this release is for you.

---

## What‚Äôs fixed

### Windows hook ‚Üí bridge (primary fix)

On some Windows setups, Cursor runs project hooks in a sandbox that **blocks `http://127.0.0.1`**. Symptoms:

- Hooks Execution Log shows `beforeSubmitPrompt`, `preToolUse`, etc.
- `%USERPROFILE%\.stayon\hook.log` shows `post failed event=beforeSubmitPrompt port=3847`
- StayOn output stays at `StayOn activated` during Agent (no `bridge event: busy_start`)
- Manual POST / **Test Hook Bridge** still works (those paths are not sandboxed)

**v0.1.2 behavior:**

1. Hook tries HTTP POST (ports from `~/.stayon/port`, `.cursor/stayon-port`, then 3847‚Äì3852).
2. If HTTP fails ‚Üí writes event JSON to `%USERPROFILE%\.stayon\inbox\`.
3. Extension polls the inbox every 250ms and drives the same busy state machine.

Success in hook.log:

```text
inbox event=beforeSubmitPrompt (http blocked: ‚Ä¶)
```

or

```text
ok event=beforeSubmitPrompt port=3847
```

### Hook install reliability

- **Absolute Node path** in `.cursor/hooks.json` (Cursor hooks often lack shell `PATH`).
- **5s timeout** per hook entry in `hooks.json`.
- Extension writes **`.cursor/stayon-port`** on activate so hooks can find the bridge port for the open workspace.

### Hook logging noise

- Empty stdin (Cursor probe runs) is ignored instead of spamming `ERROR invalid JSON on stdin`.
- Failed POSTs log the underlying error when available.

### Setup UX

- Clearer post-setup message: self-test passing does **not** guarantee Agent hooks reach the bridge ‚Äî check hook.log after a real Agent prompt.
- Beta guide troubleshooting updated for Windows paths and inbox fallback.

## Upgrade steps (required)

1. Install **`stayon-0.1.2.vsix`** (replace 0.1.0 / 0.1.1).
2. **Reload Cursor**.
3. Open your project folder.
4. Run **StayOn: Set Up** (refreshes `.cursor/hooks/stayon-event.js` and `hooks.json`).
5. Submit a **Cursor Agent** prompt ‚Äî panel should open with **‚ÄúCursor is working‚Äù**.

Verify on Windows:

```powershell
Get-Content "$env:USERPROFILE\.stayon\hook.log" -Tail 5
```

StayOn output should show `bridge event: busy_start` and `Agent busy (hook)`.

---

## Known limitations

- **Test Hook Bridge** still bypasses Cursor‚Äôs hook runner; use a real Agent prompt to validate production behavior.
- **`ERROR invalid JSON on stdin`** may still appear in old hook.log lines from v0.1.1; upgrade + Set Up clears this going forward.
- Duplicate hooks in both `<project>/.cursor/hooks.json` and `~/.cursor/hooks.json` can cause extra empty runs ‚Äî keep StayOn hooks in the project only.
